Arbetsprocess

Del 1 - Setup:

  • Skapa ett npm-projekt (npm init) för vald katalog, då genereras package.json i VSC, med inställningar för npm-projektet.
  • Installera Parcel (install parcel --save-dev), då skapas en dev-dependency som lagras i package.jsonfilen, där alla paket hamnar i katalogen node_modules.
  • node_modules läggs till i .gitignore för att undvika versionshantering.
  • Pga dev-dependency kan projektet klonas etc. genom (npm install).
  • Lägg till startsida i package.json, samt kommandon för start och build, då finns genvägar (npm start/npm build) för att snabbt paketera ihop för automatisk publicering.
  • Underkataloger till src skapas för html, css, bilder och javascript.

Del 2 - Skapande av hemsida:

  • Verifiera i html-filer att sökvägar till css, js och images är korrekta.
  • Lägg till type="module" för javascript i index.html så att Parcel kan göra ett paket av allt och optimera processen automatiskt.
  • Lägg till relevanta delar för att skapa en responsiv hemsida (inkl. undersidor) - under utvecklingens gång används en live server som fås genom (npm run start), då kommer ändringar synas direkt utan att omladdning behövs, dessutom är det inte filerna i src som körs, utan optimerade filer som ligger i dist-katalogen.
  • Git-repo används under utvecklingens gång för versionshantering.

Del 3 - Publicering av hemsida:

  • Netlify används för publicering, där (npm run build) används.
  • Genom att välja deploy så knyts git-repot till Netlify som ligger och lyssnar på förändringar och automatiskt gör ett nytt bygge vid varje commit.

Verktyg för automatisering

  • Node.js:
    • https-förfrågningar
    • databasoperationer
    • asynkrona operationer
    • kodoptimering
  • Node.js + Parcel:
    • byggflöden
  • Netlify:
    • publicering av repo
    • skapar domän
